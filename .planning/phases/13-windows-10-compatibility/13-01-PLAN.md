  <name>Task 1: Create compat.py module + unit tests</name>
  <files>src/sip_engine/compat.py, tests/test_compat.py</files>
  <action>
Create `src/sip_engine/compat.py` with these 4 functions following the patterns in 13-RESEARCH.md:

1. **`safe_rename(src: Path, dest: Path, retries: int = 3, delay: float = 0.5) -> None`**
   - On Windows (`sys.platform == "win32"`): if `dest.exists()`, call `dest.unlink()` before renaming
   - Catch `PermissionError` and retry with exponential backoff (`delay * (attempt + 1)`)
   - On Unix: just call `src.rename(dest)` directly (atomic overwrite)
   - Re-raise `PermissionError` after final retry

2. **`count_lines(path: Path) -> int`**
   - On non-Windows: try `wc -l` via `subprocess.run` with `timeout=30` (existing pattern from loaders.py)
   - On Windows OR if `wc -l` fails: pure-Python fallback opening file in binary mode `path.open("rb")` and counting with `sum(1 for _ in f)`
   - Return 0 on any `OSError`

3. **`ensure_utf8_console() -> None`**
   - Only run on Windows (`sys.platform == "win32"`)
   - For both `sys.stdout` and `sys.stderr`: check if encoding is not UTF-8
   - If not UTF-8, call `stream.reconfigure(encoding="utf-8")`
   - Catch `AttributeError` and `OSError` silently (safety for unusual environments)

4. **`supports_unicode_blocks() -> bool`**
   - Check `os.environ.get("WT_SESSION")` — Windows Terminal always supports Unicode
   - Check if `sys.stdout.encoding` contains "utf" (case-insensitive)
   - Return `False` otherwise (triggers ASCII fallback)

Then create `tests/test_compat.py` with tests:
- `test_safe_rename_basic`: create temp file, rename to new path, verify content
- `test_safe_rename_overwrites_existing`: create src + dest, rename src→dest, verify dest has src content
- `test_safe_rename_permission_error_retries`: mock `Path.rename` to raise `PermissionError` twice then succeed, verify retries happen (use `unittest.mock.patch`)
- `test_count_lines_pure_python`: create a temp file with 5 lines, mock `sys.platform` to `"win32"`, verify returns 5
- `test_count_lines_empty_file`: verify returns 0 for empty file
- `test_ensure_utf8_console_noop_on_unix`: mock `sys.platform` to `"linux"`, verify stdout not reconfigured
- `test_supports_unicode_blocks_wt_session`: mock `WT_SESSION` env var, verify returns True
- `test_supports_unicode_blocks_utf8_encoding`: mock stdout encoding to "utf-8", verify returns True
- `test_supports_unicode_blocks_cp1252`: mock stdout encoding to "cp1252", clear WT_SESSION, verify returns False

Use `tmp_path` (pytest fixture) for all file-based tests. Use `unittest.mock.patch` for platform/env mocking.
