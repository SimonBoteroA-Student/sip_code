## Goal

Eliminate the idle-GPU pattern observed during CUDA XGBoost HP search (2-3% utilization between fits, spikes to ~90% during fits). Pre-build fold `xgb.DMatrix` objects once before the HP loop and reuse across all iterations, replacing the internal numpy→DMatrix conversion done on every `XGBClassifier.fit()`. Add `max_bin=512` to CUDA device kwargs for more GPU work per split. CPU path (non-CUDA) is unchanged.

## must_haves

- [ ] `_cv_score_scale_pos_weight()` and `_cv_score_upsampling()` accept optional pre-built `fold_dmats` parameter (list of `(dtrain, dval)` tuples) and use `xgb.train()` when provided
- [ ] `_hp_search()` pre-builds fold DMatrix objects for both strategies when `device='cuda'`; CPU path uses existing sklearn `XGBClassifier.fit()` path unchanged
- [ ] `max_bin=512` added to `device_kwargs` when `device='cuda'` in `get_xgb_device_kwargs()` or in `_hp_search()` directly
- [ ] GPU→CPU fallback path (`_train_with_fallback`) still works correctly
- [ ] CPU path produces identical CV scores (determinism check)
- [ ] DMatrix caching is CUDA-only — ROCm path and CPU path use existing code

<task id="17-04-02" title="Pre-build DMatrix folds in _hp_search() for CUDA" wave="2" depends_on="17-04-01" files="src/sip_engine/classifiers/models/trainer.py">
In `_hp_search()`, before the main `for i, params in enumerate(param_samples):` loop, add a DMatrix pre-build block that runs only when `device_kwargs.get('device', '').startswith('cuda')`.

**Pre-build logic:**
```python
fold_dmats_spw: list[tuple] | None = None
fold_dmats_ups: list[tuple] | None = None

if device_kwargs.get("device", "").startswith("cuda"):
    import xgboost as xgb
    cv = StratifiedKFold(n_splits=n_splits, shuffle=True, random_state=seed)
    fold_dmats_spw = []
    fold_dmats_ups = []
    for train_idx, val_idx in cv.split(X, y):
        X_tr, y_tr = X[train_idx], y[train_idx]
        X_val, y_val = X[val_idx], y[val_idx]
        dtrain_spw = xgb.DMatrix(X_tr, label=y_tr)
        dval = xgb.DMatrix(X_val, label=y_val)
        fold_dmats_spw.append((dtrain_spw, dval, y_val))  # keep y_val for roc_auc_score

        # Strategy B: upsampled training fold (same seed = deterministic)
        n_maj = int((y_tr == 0).sum())
        n_target = int(n_maj * 0.25 / 0.75)
        X_min = X_tr[y_tr == 1]
        if len(X_min) > 0 and n_target > 0:
            X_min_up = resample(X_min, n_samples=max(n_target, 1), replace=True, random_state=seed)
            y_min_up = np.ones(len(X_min_up), dtype=y.dtype)
            X_tr_up = np.vstack([X_maj_rows := X_tr[y_tr == 0], X_min_up])
            y_tr_up = np.concatenate([np.zeros(len(X_maj_rows), dtype=y.dtype), y_min_up])
            dtrain_ups = xgb.DMatrix(X_tr_up, label=y_tr_up)
        else:
            dtrain_ups = dtrain_spw  # fallback: same as spw
        fold_dmats_ups.append((dtrain_ups, dval, y_val))
```

Pass `fold_dmats=fold_dmats_spw` to `_cv_score_scale_pos_weight()` and `fold_dmats=fold_dmats_ups` to `_cv_score_upsampling()` inside `_compare_strategies()` (or pass directly through the comparison call chain).

Also add `max_bin=512` to `device_kwargs` when `device='cuda'`:
```python
if device_kwargs.get("device", "").startswith("cuda") and "max_bin" not in device_kwargs:
    device_kwargs = {**device_kwargs, "max_bin": 512}
```

Note: `_compare_strategies()` must be updated to accept and forward `fold_dmats_spw`/`fold_dmats_ups` to the scoring functions.

## Notes

- This plan is parallel-safe with Plan 17-03 (touches only `trainer.py`, no overlap with label/IRIC/features files)
- DMatrix pre-build adds ~200MB to RAM budget per fold set (2 strategies × 5 folds = 10 DMatrix objects × ~20MB each for this dataset size) — acceptable
- ROCm path (`device='rocm'`) does NOT get DMatrix caching — ROCm XGBoost DMatrix support is untested; keep CPU path for ROCm
- The `_fit_and_compute_test_stats()` function (called for show_stats mode) is NOT changed — it uses `XGBClassifier` directly and is called rarely (only when best score improves)
