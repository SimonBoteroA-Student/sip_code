**1. Create `src/sip_engine/ui/progress.py`** — live resource monitoring during training.

Create `TrainingProgressDisplay` class that wraps `rich.Live`:

```python
class TrainingProgressDisplay:
    """Live training progress with resource monitoring.
    
    Usage:
        display = TrainingProgressDisplay(total_iterations=200)
        display.start()
        for i in range(200):
            # ... training iteration ...
            display.update(iteration=i, best_score=0.85)
        display.stop()
    
    Also works as context manager:
        with TrainingProgressDisplay(total_iterations=200) as display:
            for i in range(200):
                display.update(iteration=i, best_score=0.85)
    """
```

**Layout structure using `rich.Layout`:**
```
╭─ HP Search Progress ─────────────────────────────╮
│  ⣾ HP Search  [████████████░░░░░░░░] 60%  ETA 3m │
╰───────────────────────────────────────────────────╯
╭─ Resources ───────────────────────────────────────╮
│  CPU:  45.2%                                      │
│  RAM:  8.3 GB / 16.0 GB (51.9%)                  │
│  GPU:  N/A                                        │
╰───────────────────────────────────────────────────╯
╭─ Best Score ──────────────────────────────────────╮
│  Best AUC-ROC: 0.8534  (iter 47/200)             │
│  Trend: ↑ improving (+0.0023 last 10 iters)      │
╰───────────────────────────────────────────────────╯
```

**Class implementation:**

- `__init__(self, total_iterations: int, model_id: str = "", device: str = "cpu")`:
  - Store total, create `rich.Progress` with SpinnerColumn, TextColumn, BarColumn, percentage, TimeRemainingColumn
  - Create Layout with three sections: progress, resources, best_score
  - Track `_best_score`, `_best_iter`, `_score_history` (last 10 scores for trend)

- `start(self)`: Initialize `rich.Live` display, start the live context

- `update(self, iteration: int, best_score: float | None = None)`:
  - Update progress bar to `iteration`
  - Update resource panel:
    - `psutil.cpu_percent(interval=None)` — non-blocking (returns last reading)
    - `psutil.virtual_memory()` — current RAM usage
    - GPU utilization via `pynvml` if available and device != 'cpu', else "N/A"
  - Update best score panel if `best_score` is provided and higher than current best
  - Calculate trend: compare last 10 scores, show ↑ improving / → stable / ↓ declining
  - **IMPORTANT:** Call `psutil.cpu_percent(interval=None)` not `interval=1` — interval=1 blocks for 1 second per call which would destroy performance

- `stop(self)`: Stop `rich.Live`, print final summary

- `__enter__` / `__exit__`: Context manager support

**Non-intrusive resource monitoring (per user's specific idea):**
The resource stats panel stays at a fixed position — `rich.Live` handles this automatically. Set `refresh_per_second=4` for smooth updates without flooding output.

**Best-so-far trend calculation:**
```python
def _calculate_trend(self) -> str:
    if len(self._score_history) < 2:
        return ""
    recent = self._score_history[-10:]
    if len(recent) >= 2:
        diff = recent[-1] - recent[0]
        if diff > 0.001:
            return f"↑ improving (+{diff:.4f} last {len(recent)} iters)"
        elif diff < -0.001:
            return f"↓ declining ({diff:.4f} last {len(recent)} iters)"
    return "→ stable"
```
