Create `src/sip_engine/models/trainer.py` with the following functions. Import FEATURE_COLUMNS from `sip_engine.features.pipeline`.

**Constants:**
- `PARAM_DIST` — Gallego et al. (2021) HP search space per RESEARCH.md:
  ```python
  PARAM_DIST = {
      'n_estimators': stats.randint(50, 501),
      'max_depth': stats.randint(3, 8),
      'learning_rate': stats.loguniform(0.01, 0.3),
      'subsample': stats.uniform(0.5, 0.5),
      'colsample_bytree': stats.uniform(0.5, 0.5),
      'min_child_weight': stats.randint(1, 11),
      'gamma': [0, 0.1, 0.5, 1.0],
      'reg_alpha': [0, 0.1, 1.0],
      'reg_lambda': [0, 1, 5],
  }
  ```
- `MODEL_IDS = ["M1", "M2", "M3", "M4"]`
- `RANDOM_SEED = 42`

**Functions:**

1. `_detect_xgb_device() -> dict` — Returns XGBoost device kwargs. Check nvidia-smi subprocess for CUDA; fall back to `{'tree_method': 'hist'}` for CPU. Apple Silicon (ARM64 Darwin) uses CPU (no MPS support in XGBoost). Per RESEARCH.md Pattern 5.

2. `_stratified_split(X: pd.DataFrame, y: pd.Series, test_size: float = 0.3, seed: int = RANDOM_SEED) -> tuple[pd.DataFrame, pd.DataFrame, pd.Series, pd.Series]` — Wraps `sklearn.model_selection.train_test_split(X, y, test_size=test_size, stratify=y, random_state=seed)`. Returns (X_train, X_test, y_train, y_test). MODL-07 implementation.

3. `_cv_score_scale_pos_weight(params: dict, X: np.ndarray, y: np.ndarray, n_splits: int = 5, seed: int = RANDOM_SEED, device_kwargs: dict | None = None) -> tuple[float, float]` — Manual CV loop for Strategy A. Compute `scale_pos_weight = (y == 0).sum() / max((y == 1).sum(), 1)`. For each StratifiedKFold split, create XGBClassifier with params + scale_pos_weight + device_kwargs + `objective='binary:logistic'` + `verbosity=0` + `random_state=seed`. Fit on fold train, predict_proba on fold val, compute roc_auc_score. Return (mean, std) across folds.

4. `_cv_score_upsampling(params: dict, X: np.ndarray, y: np.ndarray, n_splits: int = 5, seed: int = RANDOM_SEED, device_kwargs: dict | None = None) -> tuple[float, float]` — Manual CV loop for Strategy B. Same as above, except before fitting: separate majority (y==0) and minority (y==1) in fold train, compute `n_target = int(n_maj * 0.25 / 0.75)`, use `sklearn.utils.resample(X_min, n_samples=max(n_target, 1), replace=True, random_state=seed)` to upsample minority. Concatenate with majority, fit, score on original val fold (NO upsampling). Return (mean, std).

5. `_compare_strategies(params: dict, X: np.ndarray, y: np.ndarray, n_splits: int = 5, seed: int = RANDOM_SEED, device_kwargs: dict | None = None) -> dict` — Run both `_cv_score_scale_pos_weight` and `_cv_score_upsampling` with the same params. Return dict:
   ```python
   {
       "scale_pos_weight": {"mean_cv_auc": float, "std_cv_auc": float},
       "upsampling_25pct": {"mean_cv_auc": float, "std_cv_auc": float},
       "winner": "scale_pos_weight" | "upsampling_25pct"
   }
   ```
   Winner = strategy with higher mean_cv_auc. Ties go to scale_pos_weight (simpler).

6. `_hp_search(X: np.ndarray, y: np.ndarray, n_iter: int = 200, n_splits: int = 5, seed: int = RANDOM_SEED, n_jobs: int = -1, device_kwargs: dict | None = None, progress: bool = True) -> dict` — Core HP search implementing MODL-05 + MODL-06. Steps:
   - Generate `param_samples = list(ParameterSampler(PARAM_DIST, n_iter=n_iter, random_state=seed))`
   - For EACH param set (with tqdm progress bar if progress=True): call `_compare_strategies(params, X, y, n_splits, seed, device_kwargs)`
   - Track best overall (strategy + params) by highest mean_cv_auc
   - Note: n_jobs is reserved for future parallel execution; current implementation is sequential (manual CV loop doesn't support joblib parallelism cleanly with upsampling). Document this.
   - Return dict:
     ```python
     {
         "best_params": dict,
         "best_strategy": str,
         "best_cv_auc_mean": float,
         "best_cv_auc_std": float,
         "all_results": list[dict],  # all n_iter results with params, strategy scores
         "n_iter": int,
         "n_splits": int,
     }
     ```

**Anti-patterns to avoid (from RESEARCH.md):**
- Do NOT use `use_label_encoder=False` (removed in XGBoost 2.0+)
- Do NOT use `gpu_hist` as tree_method (deprecated in XGBoost 3.x)
- Do NOT use `early_stopping_rounds` (Gallego approach: n_estimators is a search param)
- n_estimators must be set explicitly (default is None in XGBoost 3.x)

**models/__init__.py:** Update to re-export public symbols: `from sip_engine.models.trainer import train_model, MODEL_IDS` (train_model will be added in Plan 07-02; for now just export MODEL_IDS and any infrastructure symbols useful to tests).

Use standard project logging pattern: `logger = logging.getLogger(__name__)`.
