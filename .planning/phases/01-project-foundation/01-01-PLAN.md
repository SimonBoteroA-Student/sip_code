  <name>Task 1: Install Python 3.12 via pyenv and create verified venv</name>
  <files>.python-version</files>
  <action>
1. Install Python 3.12.12 via pyenv:
   ```bash
   pyenv install 3.12.12
   ```
   NOTE: This compiles from source on macOS and may take 5-10 minutes. This is expected.

2. Create `.python-version` file in project root containing `3.12.12` (locks pyenv to this version for the project).

3. Delete the existing `.venv` directory (it is Python 3.14.3, which is incompatible with XGBoost/SHAP wheels):
   ```bash
   rm -rf .venv
   ```
   IMPORTANT: Only delete `.venv` — never touch global Python installations.

4. Create a new venv using the pyenv-provided Python 3.12:
   ```bash
   ~/.pyenv/versions/3.12.12/bin/python -m venv .venv
   ```

5. Verify the venv Python version:
   ```bash
   .venv/bin/python --version  # Must show Python 3.12.12
   ```
  </action>
  <verify>
    <automated>.venv/bin/python --version 2>&1 | grep -q "3.12" && echo "PASS: Python 3.12" || echo "FAIL: Wrong Python version"</automated>
    <sampling_rate>run after this task commits, before next task begins</sampling_rate>
  </verify>
  <done>Python 3.12.12 installed via pyenv, .python-version file created, old .venv deleted, new .venv created with Python 3.12.12 confirmed.</done>

1. Create the full directory scaffold:
   ```
   src/sip_engine/          (main package)
   src/sip_engine/config/
   src/sip_engine/data/
   src/sip_engine/features/
   src/sip_engine/models/
   src/sip_engine/iric/
   tests/
   artifacts/models/
   artifacts/evaluation/
   artifacts/rcac/
   artifacts/features/
   artifacts/iric/
   ```

2. Create `__init__.py` files:
   - `src/sip_engine/__init__.py`: Contains `__version__ = "0.1.0"` and a module docstring
   - All submodule `__init__.py` files: Empty or single-line docstring
   - `tests/__init__.py`: Empty
   - Include `py.typed` marker in `src/sip_engine/` (zero cost, signals typed package)

3. Create `.gitkeep` files in each `artifacts/` subdirectory (so empty dirs are tracked by git).

4. Create `src/sip_engine/__main__.py` with argparse-based CLI stub:
   - Program name: `python -m sip_engine`
   - Description: "SIP -- Sistema Inteligente de Prediccion"
   - Subcommands (stubs only): `build-rcac`, `train`, `evaluate`, `run-pipeline`
   - If no command provided, print help and exit 0
   - If command provided, print "{command} not yet implemented." and exit 0

5. Create `pyproject.toml`:
   ```toml
   [build-system]
   requires = ["setuptools>=68"]
   build-backend = "setuptools.backends._legacy:_Backend"

   [project]
   name = "sip-engine"
   version = "0.1.0"
   requires-python = ">=3.12"
   dependencies = [
       "pandas>=2.2",
       "numpy>=1.26",
       "xgboost>=2.0",
       "scikit-learn>=1.5",
       "shap>=0.46",
       "scipy>=1.13",
       "joblib>=1.4",
       "Unidecode>=1.3",
       "openpyxl>=3.1",
       "pdfplumber>=0.11",
   ]

   [project.optional-dependencies]
   dev = [
       "pytest>=8.0",
       "ruff>=0.4",
       "pytest-cov>=5.0",
   ]

   [project.scripts]
   sip-engine = "sip_engine.__main__:main"

   [tool.setuptools.packages.find]
   where = ["src"]

   [tool.ruff]
   line-length = 100
   select = ["E", "F", "I"]
   ```
   NOTE: Use `setuptools.backends._legacy:_Backend` as the build backend (correct for modern setuptools). If pip install fails with this, fall back to `"setuptools.build_meta"`.

6. Update `.gitignore` — append these entries if not already present:
   ```gitignore
   # Generated artifacts
   artifacts/
   !artifacts/**/.gitkeep

   # Python build artifacts
   *.egg-info/
   dist/
   build/

   # Environment files
   .env
   .env.*
   ```
   Do NOT duplicate entries already in .gitignore (`__pycache__/`, `*.pyc`, `.venv/` are already there).

7. Install the package in editable mode with dev dependencies:
   ```bash
   .venv/bin/pip install -e ".[dev]"
   ```

8. Verify critical imports:
   ```bash
   .venv/bin/python -c "import xgboost; import shap; print('xgboost', xgboost.__version__, 'shap', shap.__version__)"
   ```
   This MUST succeed without errors. If it attempts to compile from source, something is wrong.

9. Verify entry point works:
   ```bash
   .venv/bin/python -m sip_engine --help
   ```

10. Verify editable install:
    ```bash
    .venv/bin/pip show sip-engine
    ```
