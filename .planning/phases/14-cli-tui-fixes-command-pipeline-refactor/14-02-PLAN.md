<objective>
Create a pipeline coordinator module and refactor the CLI so `run-pipeline` delegates to per-step
`run()` functions, eliminating the ~120 lines of duplicated inline logic in `__main__.py`.

Purpose: The current `__main__.py` is a 584-line monolith where `run-pipeline` (lines 445-563)
manually calls each builder/trainer with duplicated import+config+error handling. Extracting to
a coordinator enables: (1) single source of truth for step execution, (2) `--start-from` resume,
(3) `--force` all-steps override, (4) both `run-pipeline` and standalone commands share the same
`run()` functions.

Output: New `pipeline.py` coordinator + slimmed `__main__.py` + unit tests.
</objective>

  <name>Task 1: Create pipeline.py coordinator with PipelineConfig and step functions</name>
  <files>src/sip_engine/pipeline.py, tests/classifiers/test_pipeline.py</files>
  <action>
Create `src/sip_engine/pipeline.py` with the following structure:

1. **`PipelineConfig` frozen dataclass** with fields:
   - `n_jobs: int` — CPU cores
   - `n_iter: int` — HP search iterations
   - `cv_folds: int` — cross-validation folds
   - `max_ram_gb: int` — RAM limit in GB
   - `device: str` — training device (cpu/cuda/rocm)
   - `force: bool = False` — rebuild all artifacts
   - `model: str | None = None` — single model or None for all 4
   - `quick: bool = False` — quick training mode
   - `disable_rocm: bool = False` — skip ROCm GPU
   - `show_stats: bool = True` — show live test-set metrics

2. **`STEP_NAMES`** tuple: `("rcac", "labels", "features", "iric", "train", "evaluate")`

3. **Six `run_*()` functions**, each accepting `cfg: PipelineConfig` and returning `Path | list[Path]`:

   - `run_rcac(cfg)` — lazy imports `build_rcac` from `sip_engine.shared.data.rcac_builder`, calls `build_rcac(force=cfg.force)`
   - `run_labels(cfg)` — lazy imports `build_labels` from `sip_engine.shared.data.label_builder`, calls `build_labels(force=cfg.force)`
   - `run_features(cfg)` — lazy imports `build_features` from `sip_engine.classifiers.features.pipeline`, calls `build_features(force=cfg.force, n_jobs=cfg.n_jobs, max_ram_gb=cfg.max_ram_gb, device=cfg.device, interactive=False, show_progress=True)`
   - `run_iric(cfg)` — lazy imports `build_iric` from `sip_engine.classifiers.iric.pipeline`, calls `build_iric(force=cfg.force)`
   - `run_train(cfg)` — lazy imports `train_model, MODEL_IDS` from `sip_engine.classifiers.models.trainer`. Iterates over `[cfg.model] if cfg.model else MODEL_IDS`. Calls `train_model(model_id=mid, force=cfg.force, quick=cfg.quick, n_iter=cfg.n_iter, n_jobs=cfg.n_jobs, device=cfg.device, disable_rocm=cfg.disable_rocm, interactive=False, show_stats=cfg.show_stats)`. Returns list of result paths.
   - `run_evaluate(cfg)` — lazy imports `evaluate_all, evaluate_model, MODEL_IDS` from evaluator. Dispatches single model vs all based on `cfg.model`.

   **CRITICAL:** All domain module imports MUST be lazy (inside function body) to prevent circular imports. This matches the existing pattern in `__main__.py`.

4. **`run_pipeline(cfg, start_from=None)` orchestrator**:
   - Validates `start_from` against `STEP_NAMES`, raises `ValueError` with helpful message if invalid
   - Slices `STEP_NAMES` from `start_from` index if provided
   - Creates a `Rich Console`, prints config summary banner
   - Iterates steps, printing step banners via `console.rule(f"[bold]{label}")` using:
     ```python
     _STEP_LABELS = {
         "rcac": "[1/6] RCAC",
         "labels": "[2/6] Labels",
         "features": "[3/6] Features",
         "iric": "[4/6] IRIC Scores",
         "train": "[5/6] Training Models",
         "evaluate": "[6/6] Evaluation",
     }
     ```
   - Calls the corresponding `run_*()` function for each step
   - If `start_from` is used, print a warning: `"⚠ Starting from '{start_from}' — assumes earlier steps have been run"`
   - After train step, prints success line for each model: `"  ✓ Model {mid} trained: {path}"`
   - After evaluate step, prints success line: `"  ✓ Evaluation complete: {path}"`
   - After all steps, prints completion panel: `Panel("  All stages completed successfully.", title="Pipeline Complete", border_style="bright_green")`

5. **`_STEP_FNS` dict** mapping step name → function, used internally by `run_pipeline`.

Also create `tests/classifiers/test_pipeline.py` with unit tests:
- Test `PipelineConfig` creation with defaults and custom values
- Test `PipelineConfig` is frozen (immutable)
- Test `STEP_NAMES` contains all 6 steps in correct order
- Test `run_pipeline` with invalid `start_from` raises `ValueError` with helpful message
- Test that all `run_*` functions exist and are callable
- Test `_STEP_LABELS` has entries for all `STEP_NAMES`
- Use `unittest.mock.patch` to mock domain module imports in run_* functions for isolated testing
