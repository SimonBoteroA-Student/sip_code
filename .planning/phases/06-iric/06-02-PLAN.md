<objective>
Implement bid distribution anomaly statistics (kurtosis and normalized relative difference) from ofertas data.

Purpose: Kurtosis and DRN are Imhof (2018) measures for detecting bid-rigging patterns. They are computed at the process level from bid amounts, then joined to contracts. These are stored in the IRIC result artifact but NOT included in the XGBoost feature vector (they are NaN-heavy due to 60%+ direct contracting).

Output: `bid_stats.py` with `compute_bid_stats()` for per-process computation and `build_bid_stats_lookup()` for offline batch streaming from ofertas CSV.
</objective>

Create `src/sip_engine/iric/bid_stats.py` with:

1. **`compute_bid_stats(bid_values: list[float]) -> dict`**:
   - Input: list of bid amounts for a single procurement process
   - Filter out NaN and non-positive values before computing
   - Compute `n_bids` = len(valid bids)
   - **Kurtosis** (`curtosis_licitacion`):
     - If `n_bids >= 4`: use `scipy.stats.kurtosis(bids, fisher=True, bias=False)` — unbiased excess kurtosis
     - If `n_bids < 4`: `float('nan')`
   - **DRN** (`diferencia_relativa_norm`):
     - If `n_bids >= 3`: sort ascending, compute `(sorted_bids[1] - sorted_bids[0]) / sorted_bids[0]`
     - If lowest bid is 0 or negative: `float('nan')` (division by zero guard)
     - If `n_bids < 3`: `float('nan')`
   - Return: `{"curtosis_licitacion": float|NaN, "diferencia_relativa_norm": float|NaN, "n_bids": int}`
   - Document the DRN formula choice explicitly in docstring: "DRN = (second_lowest - lowest) / lowest per Imhof (2018) — relative gap between the two cheapest bids; tight clustering suggests bid rigging."

2. **`build_bid_stats_lookup() -> dict[str, dict]`**:
   - Stream `load_ofertas()` from `sip_engine.data.loaders`
   - Accumulate bid values per process ID: `{proceso_id: [valor1, valor2, ...]}`
   - Use `"ID del Proceso de Compra"` as key (from OFERTAS schema) and `"Valor de la Oferta"` as bid value
   - Skip rows where `Valor de la Oferta` is NaN or non-positive
   - After streaming completes, call `compute_bid_stats()` for each process
   - Return: `{proceso_id: {"curtosis_licitacion": ..., "diferencia_relativa_norm": ..., "n_bids": ...}}`
   - Log progress: total processes found, total bids processed
   - **Memory strategy**: Only accumulate bid values (not full row dicts). After computing stats per process, discard the raw bid lists. The result dict (one entry per unique process_id with 3 floats) is much smaller than the raw 9.7M row file.

   **Important performance note**: ofertas_proceso_SECOP.csv is 3.4 GB / ~9.7M rows. The intermediate dict of lists will be large but manageable (each list element is one float). After computing stats, the final lookup dict has one entry per unique process with 3 values — this is small.
