# Milestone v1.1: Cross-Platform OS Compatibility & Windows Support

**Status:** ✅ SHIPPED 2026-03-03
**Phases:** 12–13
**Total Plans:** 7

## Overview

Extended SIP to full cross-platform compatibility (macOS/Linux/Windows/Docker) with:
- Hardware auto-detection (OS, CPU, RAM, GPU) with Intel, NVIDIA, AMD, and container support
- Interactive TUI config screen with live resource monitoring and GPU benchmarking
- GPU→CPU automatic fallback for training failures
- Windows 10 first-class support (UTF-8, safe file ops, Unicode TUI, line counting)
- GitHub Actions CI with Windows + Ubuntu matrix
- Docker support (CPU and NVIDIA CUDA images with non-root execution)

## Phases

### Phase 12: Cross-platform OS Compatibility and Training Optimization

**Goal:** Full cross-platform OS compatibility (macOS/Linux/Windows/Docker) with auto-hardware detection, interactive TUI config, GPU acceleration with fallback, and rich training progress

**Depends on:** Phase 11

**Plans:** 4 plans

Plans:

- [x] 12-01: Hardware detection foundation (OS, CPU, RAM, GPU detection + benchmark)
- [x] 12-02: Interactive TUI components (config screen + training progress)
- [x] 12-03: Training pipeline integration (wire hardware + TUI + GPU fallback + CLI flags)
- [x] 12-04: Cross-platform curl fallback + Docker support

**Details:**

**12-01 Hardware Detection:** Built HardwareConfig frozen dataclass, detect_hardware() function with container RAM/CPU detection, GPU priority chain (CUDA→Metal→ROCm→CPU), benchmark_device() for performance testing, and select_best_device() auto-selection.

**12-02 Interactive TUI:** show_config_screen() pre-training interface with hardware summary and slider configuration (CPU cores, batch size, n_iter, n_jobs). TrainingProgressDisplay with live CPU/RAM/GPU monitoring, ETA, best-score tracking. Block-style Unicode sliders (█/░) with arrow navigation and direct entry. Non-blocking resource monitoring via psutil. Non-interactive fallback for CI.

**12-03 Training Pipeline:** Hardware-aware training with auto-detection and config screen (batch mode: shown once, settings reused). GPU→CPU fallback via _train_with_fallback() for both training and CV scoring. Added --device, --disable-rocm, --no-interactive flags to train and run-pipeline. Replaced tqdm with Rich TrainingProgressDisplay.

**12-04 Docker + Curl Fallback:** Requests fallback for data downloads on systems without curl (Windows, minimal containers). CPU Docker image (Ubuntu 22.04, Python 3.12 via deadsnakes PPA) and CUDA Docker image (nvidia/cuda:12.1.0-runtime, same Python). Multi-stage builds minimize runtime. Non-root sip:1000 execution for security.

### Phase 13: Windows 10 Compatibility

**Goal:** Make SIP pipeline fully first-class on Windows 10 — fix all runtime issues (file rename, encoding, GPU detection, Unicode TUI, line counting), add GitHub Actions CI with Windows matrix, and document Windows installation

**Depends on:** Phase 12

**Plans:** 3 plans

Plans:

- [x] 13-01: Platform compat layer + consumer integration (UTF-8, safe_rename, count_lines, Unicode degradation)
- [x] 13-02: Hardware detection + benchmark Windows fixes (nvidia-smi path, ROCm guard, ThreadPoolExecutor timeout)
- [x] 13-03: GitHub Actions CI with Windows matrix + pathlib audit + README docs

**Details:**

**13-01 Platform Compat Layer:** Created compat.py with safe_rename() for Windows atomic file operations, count_lines() pure-Python fallback avoiding wc -l issues, ensure_utf8_console() for Windows console UTF-8 init, supports_unicode_blocks() for graceful degradation. Wired into loaders (downloader.py), trainer (file persistence), and ui/config.py (slider Unicode fallback). Explicit UTF-8 encoding on comparison.py.

**13-02 Hardware Detection Fixes:** Windows-compatible nvidia-smi detection with System32 fallback path loop. ROCm filesystem check guard skips on Windows. ThreadPoolExecutor timeout replaces no-op threading.Timer for cross-platform benchmark timeout support. sys.platform == 'win32' check matches CPython convention.

**13-03 GitHub Actions CI + Documentation:** Cross-platform CI matrix (Windows Server 2022 + Ubuntu 22.04, fail-fast: false for independent completion). Pathlib audit confirmed zero os.path usage in codebase. README.md Windows 10 installation section with environment setup, pip/uv install, data download, and pipeline run examples. Docker compatibility verified.

## Milestone Summary

**Key Decisions:**
- Centralized platform-specific logic in compat.py — no scattered if sys.platform guards
- Slider bar chars resolved once at widget creation (avoid repeated detection checks)
- ThreadPoolExecutor replaces threading.Timer for Windows timeout support
- Hardware detection auto-selects best device via chainable GPU priority  
- Docker uses non-root user (sip:1000) for security
- CI matrix strategy with fail-fast: false — both OS jobs complete independently
- No macOS in Windows-focused CI (not requested, saves minutes)

**Issues Resolved:**
- Windows 10 file rename race condition (safe_rename atomic swap)
- Windows 10 Unicode slider rendering (ASCII degradation with arrow markers)
- Windows 10 CUDA detection (System32 nvidia-smi fallback)
- Windows 10 benchmark timeout (ThreadPoolExecutor instead of no-op Timer)
- Cross-platform line counting (Python fallback avoiding wc -l multiline issues)

**Technical Debt Accepted:**
- Windows SMB path encoding edge cases not covered (rare in practice)
- ROCm on Windows not tested (Windows GPU support via CUDA)
