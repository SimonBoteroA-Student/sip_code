<task type="auto">
  <name>Task 2: GREEN — Implement rcac_builder.py to pass all tests</name>
  <files>src/sip_engine/data/rcac_builder.py</files>
  <action>
Create `src/sip_engine/data/rcac_builder.py` with these functions:

1. **`normalize_numero(raw: str) -> str`**: Strip all non-digit characters using `re.sub(r'[^\d]', '', str(raw))`. Return the digit-only string. Handle NaN/None by returning empty string.

2. **`is_malformed(numero: str) -> bool`**: Return True if numero is empty, all zeros (`set(numero) == {'0'}`), or `len(numero) < 3`.

3. **`normalize_tipo(raw: str | None) -> str`**: Case-insensitive mapping. Strip whitespace. Remove accents using `unicodedata.normalize('NFD', s)` + filter combining chars, then match against mapping table:
   - Contains 'CEDULA DE CIUDADANIA' (accent-stripped) -> 'CC'
   - 'CC' -> 'CC'
   - 'NIT' -> 'NIT'
   - Contains 'PERSONA' and 'JURIDICA' -> 'NIT'
   - Contains 'PERSONA' and 'NATURAL' -> 'CC'
   - Contains 'CEDULA' and 'EXTRANJERIA' -> 'CE'
   - 'PASAPORTE' -> 'PASAPORTE'
   - None / NaN / empty / everything else -> 'OTRO'

4. **`_infer_tipo(name: str | None, numero: str) -> str`**: For sources without tipo_documento (resp_fiscales, multas). Company keyword detection (case-insensitive):
   - Keywords: LTDA, SAS, S.A., S.A.S, COOPERATIVA, FUNDACION, CORPORACION, ASOCIACION, UNION TEMPORAL, CONSORCIO, E.S.P, E.S.E, E.I.C.E, SOCIEDAD
   - If any keyword in name -> 'NIT'
   - Else if len(numero) >= 9 -> 'NIT'
   - Else -> 'CC'

5. **Source extraction functions** — each returns a list of `(tipo_norm, num_norm, source_flag_name)` tuples:
   - `_extract_boletines()`: Use `load_boletines()`. Read 'tipo de documento' and 'numero de documento'. Normalize both. Source flag: 'en_boletines'.
   - `_extract_siri()`: Use `load_paco_siri()`. All tipo='CEDULA DE CIUDADANIA' -> CC. Source flag: 'en_siri'. (DATA-04)
   - `_extract_resp_fiscales()`: Use `load_paco_resp_fiscales()`. Read 'Tipo y Num Docuemento' as numero, 'Responsable Fiscal' as name. Use `_infer_tipo(name, numero)`. Source flag: 'en_resp_fiscales'. (DATA-05)
   - `_extract_multas()`: Use `load_paco_multas()`. Read `col_5` as numero, `col_6` as name. Use `_infer_tipo(name, numero)`. Source flag: 'en_multas_secop'.
   - `_extract_colusiones()`: Use `load_paco_colusiones()`. Read 'Tipo de Persona Sancionada' and 'Identificacion'. Source flag: 'en_colusiones'.

   Each extractor iterates over loader chunks, collecting tuples. No chunked processing needed (small files), but loaders are generator-based so iterate all chunks and concat.

6. **`build_rcac(force: bool = False) -> Path`**:
   - Check if `settings.rcac_path` exists and `force=False` -> log INFO "Using cached RCAC" and return path.
   - Collect all tuples from 5 extractors.
   - Group by (tipo_norm, num_norm). For each group: OR source flags, count distinct sources.
   - Build record dicts with schema from RESEARCH.md section 4 (tipo_documento, numero_documento, 6 boolean flags, num_fuentes_distintas, malformed).
   - Separate malformed records. Write them to `settings.rcac_bad_rows_path` as CSV (columns: source, tipo_documento_raw, numero_documento_raw, reason).
   - Build index: `{(tipo, num): record_dict}` for non-malformed records only.
   - Create `settings.artifacts_rcac_dir` if needed (`mkdir(parents=True, exist_ok=True)`).
   - Serialize index via `joblib.dump(index, settings.rcac_path)`.
   - Log INFO: "RCAC built -- N records, M malformed, K sources"
   - Log WARNING: "sanciones_penales_FGN has no person-level IDs -- en_sanciones_penales always False"
   - Return `settings.rcac_path`.

   **Key implementation detail for dedup:** Use a `defaultdict` keyed on `(tipo_norm, num_norm)` to collect sets of source flags. Then build final records.

   **Key implementation detail for bad rows:** During extraction, when `is_malformed(num_norm)` is True, add to a separate bad_rows list with raw values and reason. Still include in dedup (with malformed=True) so counts are accurate, but exclude from the final lookup index.

   Use `logging.getLogger(__name__)` for all logging. Use `from __future__ import annotations`.
