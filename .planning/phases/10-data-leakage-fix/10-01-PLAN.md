<objective>
Fix the duration data leakage in feature engineering by replacing the post-amendment "Fecha de Fin del Contrato" date subtraction with a text parser for the pre-amendment "Duración del contrato" column. Update all schemas, the category_b extractor, and all affected tests.

Purpose: `duracion_contrato_dias` was the #1 feature by importance — it leaked post-amendment information (contract extensions) into the model, inflating M1 AUC by ~7-15pp above honest performance. This fix ensures the temporal feature uses only pre-amendment data.

Output: Corrected schemas.py, category_b.py with `_parse_duracion_contrato()`, and all tests passing with the new duration source.
</objective>
