```python
"""Shared feature engineering pipeline for SIP.

Single code path for both offline batch processing and online per-contract
inference (FEAT-07 train-serve parity). Enforces exclusions for post-execution
variables (FEAT-08) and RCAC-derived features (FEAT-09).

Feature column order: alphabetical within each category (A → B → C).
Total: 30 features. Category D (IRIC, 4 features) added in Phase 6.
"""

# =============================================================================
# EXPLICITLY EXCLUDED — POST-EXECUTION VARIABLES (FEAT-08)
# These columns exist in the source data but MUST NOT appear in feature vectors:
# - Fecha de Inicio de Ejecucion (from contratos)
# - Fecha de Fin de Ejecucion (from contratos)
# - Valor Facturado, Valor Pagado, Valor Pendiente de Pago (payment data)
# - ALL columns from ejecucion_contratos.csv
# Rationale: SIP is an early-detection system. Post-execution data defeats
# the purpose by using information unavailable at contract signing time.
# =============================================================================

# =============================================================================
# EXPLICITLY EXCLUDED — RCAC-DERIVED FEATURES (FEAT-09)
# These features are NOT included in the XGBoost feature vector:
# - proveedor_en_rcac
# - proveedor_responsable_fiscal
# - en_siri
# - en_multas_secop
# - en_colusiones
# Rationale: RCAC is used for labels (M3/M4), background checks, and API
# response only. Including RCAC in model inputs risks circular leakage.
# =============================================================================

FEATURE_COLUMNS = [
    # Category A (10 features) — contract characteristics
    "departamento_cat", "es_contratacion_directa", "es_regimen_especial",
    "es_servicios_profesionales", "modalidad_contratacion_cat", "origen_recursos_cat",
    "tiene_justificacion_modalidad", "tipo_contrato_cat", "unspsc_categoria",
    "valor_contrato",
    # Category B (9 features) — temporal
    "dias_a_proxima_eleccion", "dias_decision", "dias_firma_a_inicio",
    "dias_proveedor_registrado", "dias_publicidad", "duracion_contrato_dias",
    "firma_posterior_a_inicio", "mes_firma", "trimestre_firma",
    # Category C (11 features) — provider/competition
    "num_actividades_economicas", "num_contratos_previos_depto",
    "num_contratos_previos_nacional", "num_ofertas_recibidas", "num_proponentes",
    "num_retrasos_previos", "num_sobrecostos_previos", "proponente_unico",
    "tipo_persona_proveedor", "valor_total_contratos_previos_depto",
    "valor_total_contratos_previos_nacional",
]

# Critical fields — rows missing ANY of these are dropped (CONTEXT.md decision)
REQUIRED_FIELDS = ["Fecha de Firma", "Valor del Contrato", "Tipo de Contrato", "Modalidad de Contratacion"]
```

**`build_features(force: bool = False) -> Path`:**

Offline batch path:
1. Check `labels.parquet` exists (required for provider history M1/M2)
2. If `features.parquet` exists and `force=False`, return early (cached)
3. Build provider history index if not already built (`build_provider_history_index()`)
4. Build procesos lookup dict: stream `procesos_SECOP.csv` using `load_procesos()`, extract only needed cols (`ID del Portafolio`, `Fecha de Publicacion del Proceso`, `Fecha de Recepcion de Respuestas`, `Fecha de Ultima Publicación`, `Respuestas al Procedimiento`, `Proveedores Unicos con Respuestas`), build `dict[portafolio_id -> row_dict]`
5. Build proveedores lookup dict: stream `proveedores_registrados.csv`, extract (`NIT`, `Fecha Creación`), build `dict[normalized_nit -> fecha_creacion_date]` (parse MM/DD/YYYY format)
6. Build `num_actividades_economicas` dict: from full contratos, group by normalized provider ID, count distinct `unspsc_categoria` segments per provider (static attribute, not as-of)
7. Stream contratos: for each row, drop if missing REQUIRED_FIELDS (log counts at INFO level), then:
   - `compute_category_a(row)` → 10 features
   - Look up procesos data via `Proceso de Compra` ↔ portafolio dict
   - Look up proveedor fecha_creacion via normalized provider doc
   - `compute_category_b(row, procesos_data, proveedor_fecha_creacion)` → 9 features
   - `lookup_provider_history(tipo_doc, num_doc, fecha_firma, departamento)` → 6 fields
   - Get `num_actividades_economicas` from precomputed dict
   - `compute_category_c(row, procesos_data, provider_history, num_actividades)` → 11 features
   - Merge all 30 features into row dict
8. Build encoding mappings from the full feature DataFrame (training-only threshold — in v1 batch mode, all data is used; Phase 7 will split before calling build_features with a train subset)
9. `apply_encoding(df, mappings)` on categorical columns
10. Select only `FEATURE_COLUMNS` in canonical order + `id_contrato` as index
11. Write to `features.parquet` via pyarrow
12. Return path

**`compute_features(contract_row: dict, as_of_date: datetime.date, procesos_data: dict | None = None, proveedor_fecha_creacion: datetime.date | None = None, num_actividades: int = 0) -> dict`:**

Online inference path — uses same `compute_category_a/b/c` functions:
1. Load encoding mappings from JSON
2. Load provider history index
3. Compute all 30 features via the same functions
4. Apply encoding using loaded mappings
5. Return dict with FEATURE_COLUMNS keys
