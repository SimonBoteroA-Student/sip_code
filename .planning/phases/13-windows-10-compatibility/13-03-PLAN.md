Create `.github/workflows/` directory if it doesn't exist, then create `ci.yml`:

```yaml
name: CI

on:
  push:
    branches: [main, master]
  pull_request:
    branches: [main, master]

jobs:
  test:
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, windows-latest]
        python-version: ["3.12"]
    runs-on: ${{ matrix.os }}
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Install uv
        uses: astral-sh/setup-uv@v4
      
      - name: Set up Python
        run: uv python install ${{ matrix.python-version }}
      
      - name: Install dependencies
        run: uv sync --dev
      
      - name: Run tests
        run: uv run pytest tests/ -v --tb=short -m "not system"
```

Key decisions per CONTEXT.md:
- **Matrix strategy** (not separate jobs) — `fail-fast: false` so both OS jobs complete
- **Full test suite** — all tests run on both platforms (`-m "not system"` only excludes tests needing real data files, which aren't available in CI)
- **uv** for dependency management — consistent with local workflow
- **Python 3.12** — matches `requires-python = ">=3.12"` in pyproject.toml
- Triggers on push/PR to main/master branches

Do NOT add macOS to the matrix (not requested, saves CI minutes).
