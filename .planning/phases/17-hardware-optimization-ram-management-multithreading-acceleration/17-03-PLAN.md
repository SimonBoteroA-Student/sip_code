## Goal

Add chunk-level multiprocessing parallelism to labels, IRIC, and features build steps using `n_jobs` from PipelineConfig. Worker processes share read-only lookup dicts via module-level globals + pool initializer pattern (cross-platform).

## must_haves

- [ ] All three build steps (labels, IRIC, features) dispatch chunks to a `multiprocessing.Pool` when `n_jobs > 1`
- [ ] Worker functions extract row-processing logic into picklable module-level functions
- [ ] Lookup dicts shared via module-level globals + pool initializer (serialize to temp pickle, workers load once at init)
- [ ] Cross-platform: works on macOS (fork), Windows (spawn), Linux (fork) — no platform-specific code paths for the pool itself
- [ ] `n_jobs=1` path runs single-process (no pool overhead) — identical output to current implementation
- [ ] MemoryMonitor from Plan 17-01/17-02 checks between chunk completions; adaptive chunk sizing controls dispatch rate
- [ ] Output determinism: same input with `n_jobs=1` vs `n_jobs=2` produces the same set of output rows (order may differ, but content identical)
- [ ] All existing tests pass; new tests for worker functions

<task id="17-03-01" title="Add pool initialization utilities to memory.py" wave="1" depends_on="" files="src/sip_engine/shared/memory.py">

Add multiprocessing support utilities to the existing `memory.py` module:

### Pool initializer pattern
```python
import tempfile
import pickle
from pathlib import Path

_shared_lookups: dict = {}

def _init_worker(lookups_path: str) -> None:
    """Pool initializer: load serialized lookups into module-global dict.
    Called once per worker process at pool creation time."""
    global _shared_lookups
    with open(lookups_path, 'rb') as f:
        _shared_lookups = pickle.load(f)

def serialize_lookups(lookups: dict, tmp_dir: str | None = None) -> str:
    """Serialize lookup dicts to a temp pickle file. Return the file path.
    Caller is responsible for cleanup (delete after pool is done)."""
    fd, path = tempfile.mkstemp(suffix='.pkl', dir=tmp_dir)
    with os.fdopen(fd, 'wb') as f:
        pickle.dump(lookups, f, protocol=pickle.HIGHEST_PROTOCOL)
    return path

def get_shared_lookups() -> dict:
    """Access the module-global shared lookups (set by pool initializer)."""
    return _shared_lookups
```

### Pool creation helper
```python
from multiprocessing import Pool

def create_worker_pool(n_jobs: int, lookups: dict) -> tuple[Pool, str]:
    """Create a multiprocessing Pool with shared lookups.
    Returns (pool, temp_pickle_path) — caller must pool.close() and delete the temp file.
    If n_jobs <= 1, returns (None, '') — caller should use single-process path."""
    if n_jobs <= 1:
        return None, ''
    lookups_path = serialize_lookups(lookups)
    pool = Pool(n_jobs, initializer=_init_worker, initargs=(lookups_path,))
    return pool, lookups_path
```

This is the cross-platform pattern: works identically on fork (macOS/Linux) and spawn (Windows). On spawn, each worker deserializes the pickle once.

3. Return `list[dict]` of result rows

### Multiprocessing dispatch
In `build_labels()`, after building lookups:
1. If `n_jobs > 1`:
   - Package lookups into a dict: `{'boletines_set': ..., 'rcac_cache': ...}`
   - Call `create_worker_pool(n_jobs, lookups)` → get pool
   - Use `pool.imap_unordered(_process_labels_chunk, chunk_generator)` to dispatch
   - Accumulate results in main process, checking MemoryMonitor between chunks
   - `pool.close(); pool.join()` in finally block; delete temp pickle file
2. If `n_jobs <= 1`: run existing single-process code path (no pool overhead)

### Integration with Plan 17-02
- MemoryMonitor checks happen in the main process loop (between `imap_unordered` results)
- Checkpoint save uses accumulated rows if abort triggered
- Lifecycle `del` for lookups happens after pool is closed
