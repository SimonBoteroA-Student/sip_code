2. **Create `src/sip_engine/iric/pipeline.py`** with:

   **`build_iric(force: bool = False) -> Path`** — batch orchestrator:
   - Check if `settings.iric_scores_path` exists and `force=False` → return early
   - Load or build IRIC thresholds:
     - If `iric_thresholds.json` does not exist (or force=True): load features.parquet (or stream contratos), call `calibrate_iric_thresholds(df)`, write to JSON
     - If exists and not forced: load from JSON via `load_iric_thresholds()`
   - Build bid stats lookup: call `build_bid_stats_lookup()` — one streaming pass over ofertas
   - Build procesos lookup dict (same pattern as `features/pipeline.py`'s `_build_procesos_lookup()` — stream `load_procesos()`, key by `ID del Portafolio`). NOTE: Factor this into a shared helper if it duplicates pipeline.py, or import from pipeline.py if the function is extractable. If not easily shareable, duplicate the lookup pattern (it's ~15 lines) with a comment referencing the original.
   - Build provider history index (call `build_provider_history_index(force=force)` or `load_provider_history_index()`)
   - Build num_actividades lookup (same pattern as pipeline.py — group contratos by normalized provider doc, count distinct UNSPSC segments)
   - Stream contratos via `load_contratos()`:
     - For each row: look up procesos_data, provider_history, bid_stats, num_actividades
     - Inject `dias_publicidad` and `dias_decision` into procesos_data if not already present (compute from Category B values or from process dates)
     - Call `compute_iric_components(row, procesos_data, provider_history, thresholds, num_actividades)`
     - Call `compute_iric_scores(components)`
     - Get bid stats from lookup: `bid_stats_lookup.get(proceso_id, {"curtosis_licitacion": float("nan"), "diferencia_relativa_norm": float("nan"), "n_bids": 0})`
     - Collect: `{id_contrato, **components (11), **scores (4), curtosis_licitacion, diferencia_relativa_norm, n_bids}`
   - Write to `iric_scores.parquet` via pyarrow
   - Return path

   **`compute_iric(contract_row: dict, procesos_data: dict | None, provider_history: dict | None, thresholds: dict, num_actividades: int = 0, bid_values: list[float] | None = None) -> dict`** — online function:
   - Call `compute_iric_components(contract_row, procesos_data, provider_history, thresholds, num_actividades)`
   - Call `compute_iric_scores(components)`
   - If `bid_values` provided: call `compute_bid_stats(bid_values)`, else kurtosis/DRN = NaN
   - Return merged dict: `{**components, **scores, **bid_stats}`
   - This is the online-path equivalent, providing train-serve parity for IRIC

Write tests (append to `tests/test_iric.py`):

- `test_build_iric_creates_parquet`: Mock loaders, verify parquet file created with expected columns (use similar pattern to test_build_features_creates_parquet from test_features.py)
