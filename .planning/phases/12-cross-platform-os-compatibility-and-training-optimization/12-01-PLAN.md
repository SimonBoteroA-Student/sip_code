**3. Create `src/sip_engine/hardware/detector.py`** with:

- `HardwareConfig` frozen dataclass with fields:
  - `os_name: Literal['Windows', 'Linux', 'Darwin']`
  - `arch: str` (e.g., 'x86_64', 'arm64', 'AMD64')
  - `cpu_cores_physical: int`
  - `cpu_cores_logical: int`
  - `ram_total_gb: float`
  - `ram_available_gb: float`
  - `gpu_type: Literal['cuda', 'metal', 'rocm', 'cpu']`
  - `gpu_available: bool`
  - `gpu_name: str | None`
  - `gpu_vram_gb: float | None`
  - `is_container: bool`

- `detect_hardware(disable_rocm: bool = False) -> HardwareConfig` function:
  - Uses `platform.system()` for OS, `platform.machine()` for arch
  - Uses `psutil.cpu_count(logical=False)` for physical cores, `psutil.cpu_count(logical=True)` for logical
  - Uses `_get_available_ram_gb()` for RAM (respects container cgroup limits)
  - Calls `_detect_gpu_type(disable_rocm)` for GPU detection
  - Detects container via `/proc/1/cgroup` or `/.dockerenv` existence

- `_get_available_ram_gb() -> float`:
  - Check cgroup v2: `/sys/fs/cgroup/memory.max`
  - Check cgroup v1: `/sys/fs/cgroup/memory/memory.limit_in_bytes`
  - Fallback: `psutil.virtual_memory().total / (1024**3)`

- `_detect_gpu_type(disable_rocm: bool = False) -> Literal['cuda', 'metal', 'rocm', 'cpu']`:
  - Priority: CUDA > Metal awareness > ROCm > CPU (per user decision)
  - `_has_cuda()`: run `nvidia-smi` subprocess with 5s timeout, check returncode == 0
  - `_has_metal()`: check `platform.system() == 'Darwin' and platform.machine() == 'arm64'` — BUT return 'cpu' because **XGBoost has NO Metal/MPS support** (per research pitfall #1). Set `gpu_type='cpu'` but still log "Apple Silicon detected, using CPU (XGBoost has no MPS support)"
  - `_has_rocm()`: check for `/opt/rocm` directory OR `ROCM_HOME` env var. If `disable_rocm=True`, skip. Log warning: "ROCm detected. If training is unstable, use --disable-rocm"
  - CPU is the fallback

- `_get_gpu_name() -> str | None`: Try `pynvml` if available, else parse first line of `nvidia-smi` output, else None

- `_get_gpu_vram_gb() -> float | None`: Try `pynvml` if available, else None

**IMPORTANT Apple Silicon handling:** Even though Metal is "detected", gpu_type MUST be 'cpu' for XGBoost. The `gpu_available` field should be False. The `gpu_name` can say "Apple Silicon (M-series) — XGBoost CPU only" for user display.

**4. Create `src/sip_engine/hardware/device.py`** with:

- `get_xgb_device_kwargs(device_type: str) -> dict`:
  - `'cuda'` → `{'device': 'cuda', 'tree_method': 'hist'}`
  - `'rocm'` → `{'device': 'cuda:0', 'tree_method': 'hist'}` (ROCm uses CUDA API in XGBoost)
  - `'cpu'` or `'metal'` → `{'tree_method': 'hist'}` (Metal has no XGBoost support)
  - This replaces the existing `_detect_xgb_device()` in trainer.py

**5. Create `src/sip_engine/hardware/benchmark.py`** with:

- `benchmark_device(device_type: str, timeout_sec: int = 10) -> float | None`:
  - Creates tiny synthetic dataset: 1000 rows, 10 features, binary target
  - Trains XGBClassifier with n_estimators=10, max_depth=3 using `get_xgb_device_kwargs(device_type)`
  - Returns elapsed time in seconds (lower = better)
  - Uses `signal.alarm` timeout on Unix; on Windows use threading.Timer
  - On timeout or error: return None (indicates device unusable)
  - Should complete in <5 seconds for any device

- `select_best_device(candidates: list[str]) -> str`:
  - Benchmarks each candidate device
  - Returns fastest. If GPU is >20% faster than CPU, prefer GPU.
  - If all benchmarks fail, return 'cpu'
  - Used during auto-detect when benchmark is enabled
