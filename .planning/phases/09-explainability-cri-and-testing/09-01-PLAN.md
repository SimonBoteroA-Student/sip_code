Create the SHAP explainability module and CRI (Composite Risk Index) computation engine.

Purpose: These are the core building blocks that the Phase 9 analyzer (Plan 09-02) will compose into the per-contract analysis function. Output: Working `explainability/` package with `shap_explainer.py` and `cri.py`, updated `model_weights.json` with risk thresholds, updated `settings.py` with SHAP artifact path, and comprehensive unit tests.

    2. Create `src/sip_engine/explainability/shap_explainer.py` (EXPL-01, EXPL-02):
       - `extract_shap_top_n(model, X_df, feature_names, n=10) -> list[list[dict]]`:
         - Uses `shap.TreeExplainer(model).shap_values(X_df)` — returns shape `(n_samples, n_features)` for XGBoost binary
         - Per row: `np.argsort(np.abs(shap_row))[::-1][:n]` to get top-N indices
         - Each entry: `{"feature": str, "shap_value": round(float(val), 6), "direction": "risk_increasing" if val > 0 else "risk_reducing", "original_value": X_df.iloc[row_idx, col_idx]}`
         - Convert numpy types to Python native (float, int, str) for JSON safety
         - Returns list of lists (one list per sample, each containing N dicts)
       - `save_shap_artifact(shap_rows: list[list[dict]], contract_ids: list[str], model_id: str, output_dir: Path | None = None) -> Path`:
         - Writes `artifacts/shap/shap_{model_id}.parquet`
         - Flatten to rows: id_contrato, rank (1-N), feature, shap_value, direction, original_value
         - Uses pyarrow for Parquet write (consistent with rest of codebase)
         - `output_dir` defaults to `get_settings().artifacts_shap_dir`
         - Create output_dir if it doesn't exist (`output_dir.mkdir(parents=True, exist_ok=True)`)

    3. Create `src/sip_engine/explainability/cri.py` (EXPL-03, EXPL-04, EXPL-05):
       - `load_cri_config(weights_path: Path | None = None) -> dict`:
         - Reads `model_weights.json` via `json.load()`
         - `weights_path` defaults to `get_settings().model_weights_path`
         - Returns full config dict (both weights and risk_thresholds)
       - `compute_cri(p_m1: float, p_m2: float, p_m3: float, p_m4: float, iric_score: float, weights: dict | None = None) -> float`:
         - If weights is None, calls `load_cri_config()` and extracts the 5 weight keys
         - CRI = w_m1*p_m1 + w_m2*p_m2 + w_m3*p_m3 + w_m4*p_m4 + w_iric*iric_score
         - Weight key mapping: "m1_cost_overruns" → p_m1, "m2_delays" → p_m2, "m3_comptroller" → p_m3, "m4_fines" → p_m4, "iric" → iric_score
         - Returns `round(float(result), 6)`
       - `classify_risk_level(cri_score: float, thresholds: dict | None = None) -> str`:
         - If thresholds is None, loads from `load_cri_config()["risk_thresholds"]`
         - Ordered check: very_low [0, 0.2), low [0.2, 0.4), medium [0.4, 0.6), high [0.6, 0.8), very_high [0.8, 1.0]
         - Boundary: inclusive lower, exclusive upper — EXCEPT very_high which includes 1.0 exactly
         - Returns title-case string: "Very Low", "Low", "Medium", "High", "Very High"
         - Use the threshold values from the config (not hardcoded) to check ranges

    4. Update `src/sip_engine/config/model_weights.json`:

         ```json
             "very_high": [0.80, 1.00]
           }
         }
         ```
       - Existing 5 weight keys unchanged — backward compatible
