<objective>
Fix TUI rendering bugs in all three config screen functions where the hardware panel is missing,
header instruction text doesn't appear, and left panel borders are clipped.

Purpose: The Rich `Group` renderable at the screen level doesn't negotiate layout dimensions in
`Live(screen=True)` alternate buffer mode, causing panels to clip, overlap, or disappear. Replacing
`Group` with `Layout` at the screen level (while keeping `Group(*lines)` inside panels) gives Rich
the explicit height/width hints it needs.

Output: Fixed `config_screen.py` with all three `_make_layout()` closures using `Layout` + verified
visually via checkpoint.
</objective>

2. **Extract a shared helper** to DRY the three identical `_make_layout()` closures:

   ```python
   def _make_screen_layout(
       hw_panel: Panel,
       config_panel: Panel,
       header_panel: Panel | None = None,
   ) -> Layout:
       """Build a full-screen Layout for Live(screen=True) rendering."""
       layout = Layout()
       parts: list[Layout] = []
       if header_panel is not None:
           parts.append(Layout(header_panel, name="header", size=5))
       # Hardware panel: 5 data lines + 2 border lines = 7, + 1 safety = 8
       parts.append(Layout(hw_panel, name="hardware", size=8))
       parts.append(Layout(config_panel, name="config"))
       layout.split_column(*parts)
       return layout
   ```

3. **Fix `show_config_screen._make_layout()`** (around line 281):
   - Build slider lines as a `list[Text]` (keep existing logic for header, slider renders, footer)
   - Instead of concatenating into a single `Text` via `append_text()`, use `Group(*lines)` as the
     Panel content: `config_panel = Panel(Group(*lines), title="Training Settings", border_style="blue")`
   - Return `_make_screen_layout(hw_panel, config_panel)` instead of `Group(hw_panel, config_panel)`
   - Change the return type annotation from `-> Group` to `-> Layout`

4. **Fix `show_features_config_screen._make_layout()`** (around line 401):
