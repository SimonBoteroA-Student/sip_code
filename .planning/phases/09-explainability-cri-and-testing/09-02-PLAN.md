    1. Create `src/sip_engine/explainability/analyzer.py` (PROJ-03):

       **`analyze_contract()` function:**
       ```python
       def analyze_contract(
           contract_row: dict,
           as_of_date: datetime.date,
           procesos_data: dict | None = None,
           proveedor_fecha_creacion: datetime.date | None = None,
           num_actividades: int = 0,
           iric_thresholds: dict | None = None,
           bid_values: list[float] | None = None,
           models_dir: Path | None = None,
           timestamp: str | None = None,
       ) -> dict:
       ```
       - Mirrors `compute_features()` signature for the first 6 params (same optional inputs pattern)
       - `models_dir` defaults to `get_settings().artifacts_models_dir`
       - `timestamp` defaults to `datetime.now(timezone.utc).isoformat()` if None — caller can freeze for determinism
       - Implementation steps:
         a. Load all 4 models + feature_registries from `models_dir/{M1,M2,M3,M4}/model.pkl` and `feature_registry.json` using `joblib.load()`
         b. Call `compute_features(contract_row, as_of_date, ...)` to get 34-feature dict
         c. For each model (M1-M4):
            - Build a 1-row DataFrame from feature dict using feature_registry column order
            - Call `model.predict_proba(X)[:, 1][0]` → P(Mi), round to 6dp
            - Call `extract_shap_top_n(model, X, feature_names, n=10)` → top-10 SHAP entries for that model
         d. Get `iric_score` from the feature dict (it's one of the 34 features)
         e. Load CRI config via `load_cri_config()`
         f. Call `compute_cri(p_m1, p_m2, p_m3, p_m4, iric_score, weights)` → CRI score
         g. Call `classify_risk_level(cri_score, thresholds)` → risk level string
         h. Build result dict matching the JSON schema from RESEARCH.md:
            ```python
            {
                "contract_id": contract_row.get("ID Contrato", contract_row.get("id_contrato", "unknown")),
                "cri": {
                    "score": round(float(cri_score), 6),
                    "level": risk_level,
                    "weights_used": {k: v for k, v in config.items() if k != "risk_thresholds"},
                },
                "models": {
                    model_id: {
                        "probability": round(float(prob), 6),
                        "shap_top10": shap_entries,
                    }
                    for model_id, prob, shap_entries in model_results
                },
                "iric_score": round(float(iric_score), 6),
                "raw_features": {k: _serialize_value(v) for k, v in feature_dict.items()},
                "metadata": {
                    "model_versions": model_versions,
                    "timestamp": timestamp,
                },
            }
            ```
         i. All floats rounded to 6dp throughout. Use a `_serialize_value(v)` helper that converts numpy types to Python native and rounds floats.

       **`serialize_to_json(result_dict: dict) -> str` function:**
       - `json.dumps(result_dict, sort_keys=True, ensure_ascii=False)`
       - This is the determinism guarantee — sorted keys + pre-rounded floats = byte-identical output

       **`_serialize_value(v)` private helper:**
       - Converts numpy int/float to Python int/float
       - Rounds floats to 6dp
       - Returns str, int, float, None, bool as-is
       - Handles numpy bool_ → Python bool

       **Model version extraction:**
       - For each model, read `training_report.json` from model dir and extract a hash/timestamp as version identifier. If file missing, use "unknown".

    2. Update `src/sip_engine/explainability/__init__.py`:
       - Add `analyze_contract` and `serialize_to_json` to imports and `__all__`
