<objective>
Implement the IRIC calculator (all 11 binary components + 4 aggregate scores) and the threshold calibration machinery.

Purpose: The IRIC is the core risk index that converts contract characteristics into 11 binary red-flag signals across 3 dimensions (competition, transparency, anomaly), aggregated into scores that become Category D model features. Threshold calibration enables percentile-based component firing segmented by contract type.

Output: `calculator.py` with `compute_iric_components()` and `compute_iric_scores()`, `thresholds.py` with `calibrate_iric_thresholds()` and lazy-loading threshold accessor, plus comprehensive TDD tests.
</objective>

1. **`calibrate_iric_thresholds(df: pd.DataFrame, min_group_size: int = 30) -> dict`**:
   - Input: DataFrame with columns `tipo_contrato`, `num_contratos_previos_nacional`, `dias_publicidad`, `dias_decision`, `valor_contrato`
   - Group by `tipo_contrato`. Types with fewer than `min_group_size` rows get merged into `"Otro"`
   - For each group (including merged "Otro"), compute percentiles P1, P5, P95, P99 for each of the 4 calibration variables
   - Return dict matching the JSON structure from research: `{"tipo_contrato": {"Prestacion de servicios": {"num_contratos_previos_nacional": {"p1": ..., "p5": ..., "p95": ..., "p99": ...}, ...}, ...}, "calibration_date": ISO8601, "n_contracts": int, "min_group_size": int}`
   - Use `np.nanpercentile()` to handle NaN values gracefully
   - Accept ANY DataFrame — do NOT hardcode loading the full dataset. This enables Phase 7 to call it with training-only data (IRIC-08)

2. **`save_iric_thresholds(thresholds: dict, path: Path | None = None) -> Path`**:
   - Write thresholds dict to JSON at `settings.iric_thresholds_path` (or provided path)
   - Create parent directory if needed
   - Return the path written

3. **`load_iric_thresholds(path: Path | None = None) -> dict`**:
   - Lazy-load from `settings.iric_thresholds_path` (or provided path) into module-level `_thresholds_cache`
   - Same caching pattern as `rcac_lookup.py` and `provider_history.py`
   - Return cached dict on subsequent calls

4. **`reset_iric_thresholds_cache() -> None`**:
   - Set `_thresholds_cache = None` for test isolation

5. **`get_threshold(thresholds: dict, tipo_contrato: str, variable: str, percentile: str) -> float | None`**:
   - Lookup `thresholds["tipo_contrato"][tipo_contrato][variable][percentile]`
   - Fallback chain: exact tipo_contrato match -> "Otro" -> VigIA hardcoded fallbacks: `{"dias_publicidad": {"p99": 14}, "dias_decision": {"p95": 43}, "num_contratos_previos_nacional": {"p95": 3}, "valor_contrato": {"p99": 500_000_000}}`
   - Return None if even fallback is missing (should not happen for known variables)

<task type="auto">
  <name>Task 2: Implement calculator.py — 11 components + 4 aggregate scores</name>
  <files>src/sip_engine/iric/calculator.py, tests/test_iric.py</files>
  <action>
Create `src/sip_engine/iric/calculator.py` with:

1. **`compute_iric_components(row: dict, procesos_data: dict | None, provider_history: dict | None, thresholds: dict, num_actividades: int = 0) -> dict`**:

   Computes all 11 binary components. Parameters:
   - `row`: contract dict (from contratos CSV) — keys include `Modalidad de Contratacion`, `TipoDocProveedor`, `Documento Proveedor`, `Valor del Contrato`, `Justificacion Modalidad de Contratacion`, `Tipo de Contrato`
   - `procesos_data`: dict from procesos lookup (or None if no process match) — keys include `Proveedores Unicos con Respuestas`
   - `provider_history`: dict from `lookup_provider_history()` (or None for new providers) — keys include `num_contratos`, `num_sobrecostos`, `num_retrasos`
   - `thresholds`: loaded IRIC thresholds dict
   - `num_actividades`: count of distinct UNSPSC segments for this provider

   **Competition dimension (6 components — IRIC-01):**
   - `unico_proponente`: 1 if `procesos_data` is not None AND `procesos_data.get("Proveedores Unicos con Respuestas", 0) <= 1`, else 0. NaN (use None) if `procesos_data is None`
   - `proveedor_multiproposito`: 1 if `num_actividades > 1`, else 0
   - `historial_proveedor_alto`: 1 if `provider_history` is not None AND `provider_history["num_contratos"] > get_threshold(thresholds, tipo_contrato, "num_contratos_previos_nacional", "p95")`, else 0. 0 if no provider history.
   - `contratacion_directa`: 1 if `modalidad in {"Contratacion directa", "Contratacion Directa (con ofertas)"}` (case-insensitive on accent variations), else 0
   - `regimen_especial`: 1 if `modalidad in {"Contratacion regimen especial", "Contratacion regimen especial (con ofertas)"}`, else 0
   - `periodo_publicidad_extremo`: 1 if `dias_publicidad == 0 OR dias_publicidad > get_threshold(thresholds, tipo_contrato, "dias_publicidad", "p99")`, else 0. Use `procesos_data.get("dias_publicidad")` or compute from process dates. None if procesos_data is None.

   **Transparency dimension (2 components — IRIC-02):**
   - `datos_faltantes`: 1 if ANY of 3 sub-checks fires:
     - `error_documento`: `TipoDocProveedor == "No Definido"` OR `len(normalize_numero(str(doc))) < 6` OR doc has no digits at all
     - `error_justificacion`: `Justificacion Modalidad de Contratacion` is None/NaN/empty or equals "no especificado" (case-insensitive, stripped)
     - `error_valor`: `Valor del Contrato > get_threshold(thresholds, tipo_contrato, "valor_contrato", "p99")`
   - `periodo_decision_extremo`: 1 if `dias_decision == 0 OR dias_decision > get_threshold(thresholds, tipo_contrato, "dias_decision", "p95")`, else 0. None if procesos_data is None (no process → decision period unknown). Use `procesos_data.get("dias_decision")`.

   **Anomaly dimension (3 components — IRIC-03):**
   - `proveedor_sobrecostos_previos`: 1 if `provider_history is not None AND provider_history["num_sobrecostos"] > 0`, else 0. **Explicitly 0 (not NaN) for providers with no history** — per VigIA pattern and IRIC success criterion 6.
   - `proveedor_retrasos_previos`: 1 if `provider_history is not None AND provider_history["num_retrasos"] > 0`, else 0. Same 0-for-new-providers rule.
   - `ausencia_proceso`: 1 if `procesos_data is None`, else 0.

   Return dict with all 11 component names as keys, values are int (0/1) or None (when data unavailable).

   **NaN handling for components:** Components 1, 6, 8 return None when procesos_data is None (the missing-process irregularity is captured by component 11). Components 9, 10 return 0 for new providers (VigIA fills NaN as 0).

2. **`compute_iric_scores(components: dict) -> dict`**:
   - Treat None values as 0 when summing (VigIA pattern: `sum(axis=1)` skips NaN)
   - `iric_score` = (1/11) * sum(all 11 components, None→0)
   - `iric_competencia` = (1/6) * sum(components 1-6, None→0)
   - `iric_transparencia` = (1/2) * sum(components 7-8, None→0)
   - `iric_anomalias` = (1/3) * sum(components 9-11, None→0)
   - Return dict with these 4 keys

   Import `normalize_numero` from `sip_engine.data.rcac_builder` for the `datos_faltantes` document validation sub-check. Import `get_threshold` from `sip_engine.iric.thresholds`.

   **Important:** `dias_publicidad` and `dias_decision` values must be PASSED IN via `procesos_data` dict (already computed by Category B in pipeline.py). Do NOT recompute them. The pipeline injects `"dias_publicidad"` and `"dias_decision"` into `procesos_data` before calling IRIC (or the caller passes pre-computed values). For the batch path, these come from the features already computed. For component firing, read from `procesos_data.get("dias_publicidad")` and `procesos_data.get("dias_decision")`.
