    **2. Add _compute_m3_m4() to label_builder.py:**

    ```python
    def _compute_m3_m4(df: pd.DataFrame, boletines_set: set[tuple[str, str]]) -> pd.DataFrame:
        """Compute M3 and M4 labels on the contratos DataFrame.

        M3: provider in boletines_set (direct query, not via RCAC).
        M4: provider found in RCAC via rcac_lookup().

        Null handling: M3=null and M4=null when provider ID is missing or malformed.
        """
    ```
    - For each row, normalize provider: `tipo_norm = normalize_tipo(row["TipoDocProveedor"])`, `num_norm = normalize_numero(row["Documento Proveedor"])`
    - Handle vectorized (use `.apply()` or vectorized string ops):
      - Create `tipo_norm` series via `df["TipoDocProveedor"].apply(normalize_tipo)`
      - Create `num_norm` series via `df["Documento Proveedor"].apply(normalize_numero)`
      - Create `malformed_mask` via `num_norm.apply(is_malformed)` OR `df["Documento Proveedor"].isna()`
    - M3 column (nullable Int8):
      - For non-malformed rows: `1` if `(tipo_norm, num_norm)` is in `boletines_set`, else `0`
      - For malformed rows: `pd.NA`
      - Use `pd.array([...], dtype="Int8")` to support nullable integers
    - M4 column (nullable Int8):
      - For non-malformed rows: `1` if `rcac_lookup(tipo_norm, num_norm)` returns a record (not None), else `0`
      - For malformed rows: `pd.NA`
      - Note: `rcac_lookup()` normalizes internally, but we already have normalized values — pass raw values to let it re-normalize (safe, idempotent)
      - Actually, pass the RAW `TipoDocProveedor` and `Documento Proveedor` directly to `rcac_lookup()` since it normalizes internally. For M3, use the pre-normalized values since we're checking set membership.
    - Log M3 summary: `logger.info("M3: %d positive (%.2f%%), %d null (malformed provider ID)", ...)`
    - Log M4 summary: `logger.info("M4: %d positive (%.2f%%), %d null (malformed provider ID)", ...)`
    - Return the updated DataFrame with M3 and M4 columns
